# 当前开发上下文

日期：2026-05-31

## 项目目标

做一个高仿口袋记账 iOS 体验、但自有风格的私有记账 PWA，部署在群晖 Container Manager，数据保存在 SQLite，支持从口袋记账导出数据迁移。

## 已完成

- 定版文档：`docs/口袋记账APP定版文档.md`
- 项目骨架：
  - `apps/api`：Fastify + SQLite/Drizzle 骨架
  - `apps/web`：React + Vite + PWA 骨架
  - `packages/shared`：共享类型
- 依赖已安装，使用项目内便携 Node：
  - `.tools/node-v24.16.0-win-x64`
- 已通过：
  - `npm run typecheck`
  - `npm run build`
- 当前服务：
  - `http://localhost:3000`
  - 生产服务进程曾使用 PID `26008`

## 已修复的前端体验

- 底部导航改为 5 栏：资产 / 报表 / 账单 / 借贷 / 更多
- 记一笔支持选择分类、金额键盘、日期/账户/成员/备注弹层
- 记一笔确认后可写入浏览器 localStorage 并回到账单首页展示
- 资产页支持选择计入资产的账户
- 报表页支持分类/趋势/对比/成员 Tab
- 修复趋势图左轴裁切

## 当前轮目标

成品化修复和群晖部署前验收：

1. 借贷影响账户余额：借出/追加借出扣账户，收款/利息收入加账户，借入/追加借入加账户，还款/利息支出扣账户。
2. 借贷流水使用 `transactions.type = loan` 并关联 `loan_id`，从普通收支、预算和报表中排除。
3. 月总预算持久化，首页预算气泡和“更多 - 预算设置”都可调整。
4. 账单时间线改成收入在左、支出在右。
5. 记账键盘贴近底部，账户编辑和借贷入口要有清晰高亮状态。
6. 按用户提供的 `DESIGN.md` 的白底、深墨蓝、电靛蓝、扁平金融风格优化前端。
7. 本机 Docker 最后用独立 `.codex_tmp/docker-data` 验证；群晖目标路径是 `/volume1/docker/pocket-ledger`。

## 重要注意

- 不要把 `.tools/`、`node_modules/`、`data/` 纳入代码交付。
- 生产依赖 audit 当前干净；此前移除了 `xlsx`，因为有无修复高危漏洞。
- 用户正式过渡前会再提供最新口袋记账导出文件，需要重新做清空导入和对账。
