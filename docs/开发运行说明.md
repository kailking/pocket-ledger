# 开发运行说明

## 环境要求

- Node.js 22+
- npm 11+
- Docker Desktop 或兼容的 Docker Engine，仅在需要本地容器构建时使用

## 安装依赖

```bash
npm install
```

## 本地环境变量

复制示例文件：

```bash
cp .env.example .env
```

本地开发默认使用：

```text
DATABASE_URL=file:./data/app.db
BACKUP_DIR=./data/backups
UPLOAD_DIR=./data/uploads
WEB_DIST_DIR=./public
```

根目录 `docker-compose.yml` 会覆盖容器内路径，让容器始终使用：

```text
DATABASE_URL=file:/data/app.db
BACKUP_DIR=/data/backups
UPLOAD_DIR=/data/uploads
WEB_DIST_DIR=/app/public
```

## 启动后端

```bash
npm run dev:api
```

后端默认监听：

```text
http://localhost:3000
```

健康检查：

```text
http://localhost:3000/health
```

## 首次登录

默认管理员账号由环境变量控制：

```text
ADMIN_USERNAME=admin
ADMIN_INITIAL_PASSWORD=change-me-on-first-login
SESSION_SECRET=development-session-secret-change-me
```

首次登录后必须修改初始密码。未登录访问业务 API 会返回 401；已登录但尚未修改初始密码时，业务 API 会返回 403。

## 启动前端

```bash
npm run dev:web
```

前端默认监听：

```text
http://localhost:5173
```

开发模式下 Vite 会把 `/api` 和 `/health` 代理到 `http://localhost:3000`。

## 本地构建

完整构建：

```bash
npm run build
```

构建顺序：

1. `packages/shared`
2. `apps/api`
3. `apps/web`

## 本地 Docker 运行

先确认 `.env` 存在，再执行：

```bash
npm run docker:up
```

默认访问：

```text
http://localhost:3456
```

常用命令：

```bash
npm run docker:build
npm run docker:config
npm run docker:logs
npm run docker:down
```

本地 Docker 数据默认挂载到：

```text
./data
```

可在 `.env` 中修改：

```text
APP_HOST_PORT=3456
DOCKER_APP_URL=http://localhost:3456
LEDGER_DATA_DIR=./data
```

## 数据库

数据库使用 SQLite。开发默认路径：

```text
./data/app.db
```

生成迁移：

```bash
npm run db:generate
```

执行迁移：

```bash
npm run db:migrate
```

## 发布前检查

建议主 Agent 在可用环境中执行：

```bash
npm install
npm run typecheck
npm run build
npm run docker:build
npm run docker:config
```
