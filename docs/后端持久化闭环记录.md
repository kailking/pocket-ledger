# 后端持久化闭环记录

日期：2026-05-31

## 本轮已完成

- 认证闭环：
  - `GET /api/auth/me`
  - `POST /api/auth/login`
  - `POST /api/auth/logout`
  - `POST /api/auth/change-password`
  - 管理员密码使用 scrypt 哈希保存。
  - 业务 API 未登录返回 401。
  - 首次登录未修改初始密码时，业务 API 返回 403，只允许先改密码。
- 口袋记账导入闭环：
  - `POST /api/imports/pocket/preview` 读取真实 `.xls`。
  - `POST /api/imports/pocket/commit?mode=clear` 支持清空当前账务数据后导入。
  - 导入会创建账户、分类、账本、成员、收支流水、转账配对、借入借出记录和导入批次。
  - 前端“数据工具”页已支持选择文件、预览、清空并导入、查看导入批次。
- 借入借出闭环：
  - `/api/loans` 已从占位数据改为 SQLite 真实数据。
  - 支持列表、详情、新增、编辑、软删除、添加还款/追加/利息流水、结清、重开。
  - 前端“借贷”页已从静态假数据改为读取 `/api/loans`，并支持新增借入/借出。
- 转账闭环：
  - 前端记账页支持转账保存、编辑、详情读取和删除。
  - `/api/transfers` 已从占位接口改为真实 SQLite 创建、列表、详情、编辑、删除。
  - 首页、搜索页、详情页均显示转账金额。
- 备份/恢复闭环：
  - `POST /api/backups/create` 生成 SQLite 备份。
  - `GET /api/backups` 列出备份。
  - `GET /api/backups/:fileName/export` 下载备份。
  - `POST /api/backups/restore` 按 `{ "fileName": "xxx.db" }` 从已有备份恢复，并在恢复前自动生成当前库留档。
  - 前端“数据工具/备份”页已支持创建、下载、恢复备份。
  - 群晖 `restore.sh` 在覆盖 `app.db` 前，会把当前 `app.db`、`app.db-wal`、`app.db-shm` 留档到 `data/backups/pre-restore-时间戳/`。
- 账户管理闭环：
  - `/api/accounts` 已支持列表、详情、新增、编辑、归档、重算、保存“计入资产”。
  - 资产页已支持新增账户、编辑账户、选择账户颜色/类型、设置初始余额、归档账户、资产/负债切换、资产趋势概览。
- 群晖部署骨架：
  - Docker 单容器构建通过。
  - SQLite、备份、上传目录统一挂载到 `/data`。
  - 群晖 compose 端口为 `3456:3000`。
  - `docs/群晖部署说明.md`、`deploy/synology/*` 已更新。

## 本轮修复补充

- 修改密码体验：
  - 前端增加新密码长度、确认密码、一致性、不能等于当前密码的校验。
  - 后端全局错误处理兼容 Zod 校验错误，参数错误返回 400，不再变成 500。
  - API 错误读取会提取后端 `details` 中的第一条校验信息。
- 大数据搜索和详情：
  - 搜索页改为调用后端筛选，默认请求 `limit=500`，不再只在前端过滤前 200 条。
  - 账单详情和编辑页改为读取 `/api/transactions/:id`，不再依赖默认列表。
- 交互安全：
  - 清空导入增加二次确认。
  - 分类删除增加二次确认。
  - 账户归档增加二次确认。
  - 导入预览、创建备份、分类保存等 pending 状态会禁用按钮。
  - 报表页不再在接口失败或空数据时展示示例假数据；对比和成员统计已接入真实接口。
  - 借贷流水删除后会回算剩余金额、本金、利息和状态；手动结清后重开会按流水恢复剩余金额。
  - Drizzle schema 补齐 `users.must_change_password`，启动时也会自动补旧库缺失列。
