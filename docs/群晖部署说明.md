# 群晖 Container Manager 部署说明

本文面向群晖 NAS 私有部署。运行形态是一个容器同时提供 API 和前端静态文件，SQLite 数据库、备份文件、上传文件全部挂载到宿主机 `/volume1/docker/pocket-ledger/data`，容器内统一使用 `/data`。

## 目录准备

在群晖上创建以下目录：

```text
/volume1/docker/pocket-ledger/
  config/
    .env
  data/
    app.db
    app.db-wal
    app.db-shm
    backups/
    uploads/
  docker-compose.yml
```

`app.db`、`app.db-wal`、`app.db-shm` 会由应用自动创建。首次部署时只需要提前创建 `config/` 和 `data/` 目录即可。

## 配置文件

把仓库里的 `deploy/synology/env.example` 复制到群晖：

```text
/volume1/docker/pocket-ledger/config/.env
```

必须修改：

```text
APP_URL=http://你的群晖IP:3456
SESSION_SECRET=至少24位随机字符串
ADMIN_INITIAL_PASSWORD=首次登录密码
```

生产部署建议保持这些路径不变：

```text
DATABASE_URL=file:/data/app.db
BACKUP_DIR=/data/backups
UPLOAD_DIR=/data/uploads
WEB_DIST_DIR=/app/public
```

这些路径是容器内路径，实际会落到群晖的 `/volume1/docker/pocket-ledger/data`。

变量说明：

| 变量 | 用途 | 群晖建议值 |
| --- | --- | --- |
| `APP_ENV` | 运行环境 | `production` |
| `APP_PORT` | 容器内监听端口 | `3000` |
| `APP_URL` | 外部访问地址，影响 CORS | `http://群晖IP:3456` |
| `DATABASE_URL` | SQLite 文件位置 | `file:/data/app.db` |
| `SESSION_SECRET` | Cookie 签名密钥 | 至少 24 位随机字符串 |
| `ADMIN_USERNAME` | 初始管理员用户名 | `admin` |
| `ADMIN_INITIAL_PASSWORD` | 初始管理员密码 | 首次登录后修改 |
| `BACKUP_DIR` | 备份输出目录 | `/data/backups` |
| `UPLOAD_DIR` | 上传文件目录 | `/data/uploads` |
| `WEB_DIST_DIR` | 前端静态文件目录 | `/app/public` |

## Compose 文件

把仓库里的 `deploy/synology/docker-compose.yml` 复制到：

```text
/volume1/docker/pocket-ledger/docker-compose.yml
```

默认端口映射：

```text
群晖 3456 -> 容器 3000
```

默认数据挂载：

```text
/volume1/docker/pocket-ledger/data -> /data
```

如果你不用 `/volume1/docker/pocket-ledger`，需要同时修改 `docker-compose.yml` 里的 `env_file` 和 `volumes` 绝对路径。

## 镜像构建

在有 Docker 的开发机或群晖 SSH 中，从仓库根目录构建：

```bash
npm run docker:build
```

等价命令：

```bash
docker build -t pocket-ledger:latest .
```

如果在开发机构建，需要导出镜像并导入群晖：

```bash
docker save pocket-ledger:latest -o pocket-ledger_latest.tar
```

把 `pocket-ledger_latest.tar` 上传到群晖后，可在 Container Manager 的“映像”页面导入，或 SSH 执行：

```bash
docker load -i pocket-ledger_latest.tar
```

## Container Manager 启动

1. 确认群晖本地已有 `pocket-ledger:latest` 镜像。
2. 确认 `/volume1/docker/pocket-ledger/config/.env` 已按上文修改。
3. 在 Container Manager 中新建项目，选择 `/volume1/docker/pocket-ledger/docker-compose.yml`。
4. 启动项目后访问：

```text
http://群晖IP:3456
```

健康检查地址：

```text
http://群晖IP:3456/health
```

## 备份

推荐在群晖 SSH 中执行在线备份脚本。它会在容器内调用 SQLite backup API，适合容器运行时备份：

```bash
sh deploy/synology/backup.sh
```

如果脚本文件已经复制到群晖项目目录，也可以在 `/volume1/docker/pocket-ledger` 下执行：

```bash
sh backup.sh
```

备份产物会出现在：

```text
/volume1/docker/pocket-ledger/data/backups/app-YYYYMMDDTHHMMSSZ.db
```

也可以使用群晖快照或文件级备份，但建议先在 Container Manager 中停止 `pocket-ledger` 容器，再备份整个目录：

```text
/volume1/docker/pocket-ledger/data
```

## 恢复

恢复前先确认要恢复的备份文件，例如：

```text
/volume1/docker/pocket-ledger/data/backups/app-20260531T120000Z.db
```

在群晖 SSH 中执行：

```bash
DATA_DIR=/volume1/docker/pocket-ledger/data sh deploy/synology/restore.sh /volume1/docker/pocket-ledger/data/backups/app-20260531T120000Z.db
```

如果脚本文件已经复制到群晖项目目录，也可以在 `/volume1/docker/pocket-ledger` 下执行：

```bash
DATA_DIR=/volume1/docker/pocket-ledger/data sh restore.sh /volume1/docker/pocket-ledger/data/backups/app-20260531T120000Z.db
```

恢复脚本会停止容器、先把当前 `app.db`/WAL/SHM 留档到 `data/backups/pre-restore-时间戳/`，再把备份覆盖到 `data/app.db`、删除旧的 `app.db-wal` 和 `app.db-shm`，最后启动容器。恢复后检查：

```text
http://群晖IP:3456/health
```

## 外网访问建议

优先使用：

- Tailscale
- ZeroTier
- 群晖 VPN

不建议直接把 `3456` 暴露到公网。
